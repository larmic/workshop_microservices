# Workshop-Feedback

Hier wird das Feedback aus durchgeführten Workshops gesammelt und in einen
umsetzbaren Backlog überführt. Ziel: nichts geht verloren, jeder Hinweis wird
verifiziert, eingeordnet und priorisiert.

## Konvention

- **Ein Ordner pro Termin:** `docs/feedback/<YYYY-MM-DD>/` (Datum des Workshops).
- **Zwei Dateien pro Termin:**
  - `feedback.md` · sachliche, anonymisierte Analyse: was gut lief, Kernbefund,
    Beobachtungen nach Themen gruppiert (je mit Verifikations-Status und
    Einordnung), plus Abgleich-Checkliste.
  - `backlog.md` · priorisierte TODO-Liste mit Checkboxen. Jeder Eintrag hat eine
    stabile ID (z. B. `A1`), betroffene Dateien, Priorität und ggf. das Kennzeichen
    *Entscheidung Lars* für inhaltliche Richtungsfragen.

## Anonymisierung

Das Repository ist öffentlich. Feedback wird daher **sinngemäß und anonymisiert**
abgelegt. Keine Klarnamen, keine wörtlichen Zitate persönlicher Rückmeldungen, nur
die sachlichen Inhalte.

## Status-Konvention im Backlog

- `[ ]` offen · `[~]` in Arbeit · `[x]` erledigt
- Priorität: **Hoch** (Lesbarkeit/Korrektheit, blockiert Verständnis) ·
  **Mittel** (klare Verbesserung) · **Niedrig** (nice to have)
- *Entscheidung Lars*: inhaltliche oder curriculare Richtungsfrage, kein reiner Fix
